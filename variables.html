  • If you need to include variable(s) in the text, following examples will help you achieve that.

  • If you want to include variable(s).

    Like user name or city...
  • javascript:
    var user_name = "Marcel";
    var city = "Cork";

    $ ( '#variable_text' ).append ( `
    <span class="___"
    data-text="This user's name is |user and he is from |city"
    data-user ="${ user_name }"
    data-city ="${ city }"
    > </span>` );
  • 1. Your named variable in data-text must be prepended with pipe | followed by named variable

    |user


    |city


    data-text ="This user's name is |user and he is from |city"

    2. You need to include actual variable data-{ named_variable } in element

    data-user = "${ user_name }"

    data-city = "${ city }"

  • Your translation file(s) must include

    |user

    |city

    in both default language and translated language.
  • assets/js/languages/en.json

    { "This user's name is |user and he is from |city" : "This user's name is |user and he is from |city" , }
  • English translation renders: This user's name is Marcel and he is from Cork
  • assets/js/languages/es.json

    { "This user's name is |user and he is from |city" : "El nombre de este usuario es |user y él es de |city" , }
  • Spanish translation renders: El nombre de este usuario es Marcel y él es de Cork

  • If you want to include variable(s) and you want to translate this variable(s).

    For example bedroom type. ( single, double ...)
  • javascript:
    var room_type = 'single';

    $ ( '#translatable_variable' ).append ( `
    <span class="___"
    data-text="This is :room_type bedroom"
    data-room_type="${ room_type }">
    </span> ` );
  • 1. Your named variable in data-text must be prepended with colon : followed by named variable

    :room_type


    data-text ="This is :room_type bedroom

    2. You need to include actual variable data-{ named_variable } in element

    data-room_type = "${ room_type }"

  • Your translation file(s) must include

    :room_type

    in both default language and translated language.
  • You need to include translated version(s) of variable(s) as well.
    For example, in this case room_type can be single or double,
    so we will include translated versions of it in our translation files.
  • assets/js/languages/en.json

    {
    "single": "single",
    "double": "double",
    "This is :room_type bedroom" : "This is :room_type bedroom" ,
    }
  • English translation renders:
    This is single bedroom
    This is double bedroom
  • assets/js/languages/es.json

    {
    "single": "individual",
    "double": "doble",
    "This is :room_type bedroom" : "Esto es habitación :room_type" ,
    }
  • Spanish translation renders:
    Esto es habitación individual
    Esto es habitación doble
